Privacy Policy

To facilitate the ability of users to vote, we create a unique 'voter ID', which is generated using the following information:

  • IP address
  • User-agent header
  • Random number generated at that instance (pepper)

An irreversable cryptographic hash is created using this information in order to roughly identify a user, allowing us to keep track of upvotes without de-anonymising users. This hash is not linked with your session.

On the complaints page, users may leave an email address if further correspondence is required. Complaints can not be linked with the session, or any post under the ticketing system.

We do log requests as you browse the services and store it for 30 days:

  • Your IP address
  • What was requested
  • Was the request a success

The purpose of this data collection is to help us identify any potential abuse.

Cookies

Session cookies ("hithereimacookie") are used to generate a temporary username, allowing a user to comment anonymously and create discussions. These sessions are set to expire in one month. This session is not linked with the voter ID.

Examples of data stored on a session:

Third Parties

There's only one party happening here. No information is shared with third parties.

Changes

We, the current class representatives, reserve the right to make any changes to this privacy policy. Notice will be given as an alert on the homepage.
